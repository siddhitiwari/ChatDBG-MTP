%% ─── Abstract ────────────────────────────────────────────────────────────────
\begin{abstract}
Debugging is one of the most time-consuming activities in software development.
Existing AI-assisted debugging tools, including ChatDBG, require an active
debugger session, limiting their applicability to crashes reproducible in a live
environment. This paper presents \textbf{ChatDBG-MTP}, which introduces five
contributions: (1)~\emph{post-mortem analysis} from crash log files,
(2)~a \emph{zero-friction exception hook} via \texttt{sys.excepthook},
(3)~\emph{repository-aware context} through import-graph traversal,
(4)~\emph{git-aware context} using \texttt{log}/\texttt{blame}/\texttt{diff},
and (5)~\emph{automated fix application and test generation}.
Evaluating on 8 benchmarks across four variants, repository context alone
improves root-cause keyword coverage from 57\,\% to 77\,\% (+20\,pp) and fix
correctness from 50\,\% to 75\,\% (+25\,pp). The full \emph{+Repo+Git}
configuration achieves 89\,\% keyword coverage and 88\,\% fix correctness at
only a 38\,\% cost increase over the baseline.
\end{abstract}

\begin{IEEEkeywords}
debugging, large language models, post-mortem analysis, automated program repair,
fault localization, git-aware debugging, test generation
\end{IEEEkeywords}

%% ─── 1. Introduction ─────────────────────────────────────────────────────────
\section{Introduction}
\IEEEPARstart{M}{odern} software crashes in many contexts where a live debugger
is unavailable: production environments, CI pipelines, and remote machines.
Even when a debugger is available, AI assistants such as
ChatDBG~\cite{chatdbg} only analyze the immediate stack trace, missing the
broader codebase context that could accelerate diagnosis.

We identify three key gaps: \emph{(i)~Availability}---tools require a live
process; \emph{(ii)~Context}---analysis is limited to the call stack, ignoring
imported modules and recent code changes; \emph{(iii)~Completeness}---tools
explain bugs but do not apply fixes or write regression tests.

ChatDBG-MTP addresses all three through: a post-mortem mode
(\texttt{chatdbg --analyze crash.log}), a zero-friction exception hook
(\texttt{chatdbg.install()}), repository context (\texttt{--repo}), git context
(automatic), and an automated repair pipeline (\texttt{apply\_fix},
\texttt{generate\_test}).

%% ─── 2. Background ────────────────────────────────────────────────────────────
\section{Background and Related Work}

\subsection{ChatDBG}
ChatDBG~\cite{chatdbg} integrates an LLM into Python \texttt{pdb/ipdb} and native
GDB/LLDB.  When the user types \texttt{why}, it sends the stack trace and locals
to the model, which can call back via \texttt{debug}, \texttt{info}, and
\texttt{slice}. ChatDBG-MTP reuses its \texttt{Assistant} class, YAML logging,
and listener infrastructure.

\subsection{Automated Program Repair}
GenProg~\cite{genprog}, Prophet~\cite{prophet}, and Angelix~\cite{angelix} apply
search or symbolic analysis to patch bugs. LLM-based successors (AlphaRepair,
ChatRepair) require a pre-existing test suite. ChatDBG-MTP operates at run time
on a crashed program with no prior test infrastructure.

\subsection{Fault Localization}
Spectrum-based FL (Tarantula~\cite{tarantula}, Ochiai) ranks suspicious
statements by test coverage. Our approach is complementary: we provide rich
context and rely on LLM reasoning.

\subsection{LLM Code Understanding}
Providing extra context---imports, docstrings, related code---improves
LLM accuracy on code tasks~\cite{codex,retrieval_prompts}. The \texttt{--repo}
flag operationalises this via automatic import-graph traversal.

\subsection{Post-Mortem Analysis}
The \texttt{postmortem} package contains: a \textbf{parser} that converts raw
tracebacks into typed \texttt{CrashReport}/\texttt{FrameInfo} dataclasses; a
\textbf{context builder} that reads source lines via \texttt{linecache},
skipping library frames; and an \textbf{analyzer} that orchestrates the pipeline
and logs to YAML.

\subsection{Exception Hook}
\texttt{chatdbg.install()} registers a \texttt{sys.excepthook} that
(1)~preserves the original traceback output, (2)~formats the exception as text,
(3)~calls \texttt{analyze\_crash\_text()} in-memory,
(4)~skips \texttt{SystemExit}/\texttt{KeyboardInterrupt}, and
(5)~swallows all ChatDBG errors to avoid masking the original exception.

\subsection{Repository Context}
\texttt{repo.py} seeds the traversal from crash-frame files, parses all
\texttt{import}/\texttt{from...import} statements with the \texttt{ast} module
to depth~2, and formats results within a $\approx$6,000-token budget.

\subsection{Git-Aware Context}
\texttt{git\_context.py} runs three commands per unique user-code file
(Table~\ref{tab:git}), each with a 10\,s timeout, failing silently when git
is absent.

\begin{table}[t]
\centering
\caption{Git Commands Collected per Crashed File}
\label{tab:git}
\small
\begin{tabular}{ll}
\toprule
\textbf{Command} & \textbf{Information} \\
\midrule
\texttt{git log --oneline -8} & Recent commit messages \\
\texttt{git blame -L s,e}     & Author/date of crash lines \\
\texttt{git diff HEAD\textasciitilde1} & What changed last commit \\
\bottomrule
\end{tabular}
\end{table}

\subsection{Automated Repair}
\textbf{\texttt{apply\_fix}}: shows a unified diff and prompts \texttt{[y/N]}
before writing. Normalises line endings for cross-platform compatibility.
\textbf{\texttt{generate\_test}}: appends a pytest test designed to fail on
buggy code and pass after the fix.
Both bypass interactive prompts in eval mode (\texttt{CHATDBG\_EVAL=1}).

\subsection{Key Findings}

\textbf{+Repo is the single most impactful feature} (+20\,pp KW, +25\,pp fix
correctness). Root causes frequently lie in imported modules not visible in the
stack trace (Fig.~\ref{fig:accuracy}).

\textbf{+Git improves recall modestly} (+15\,pp KW, +12\,pp fix) and provides the
largest gains on regression bugs where \texttt{git diff HEAD\textasciitilde1}
identifies the introducing commit.

\textbf{Combining both is super-additive} for test generation
(12\,\%\,$\to$\,88\,\%), confirming both context types are required for
runnable test output (Fig.~\ref{fig:fixtest}).

\textbf{Cost increase is modest}: +Repo+Git costs \$0.026 per analysis
(+38\,\% vs.\ Baseline) while delivering +56\,\% keyword coverage
(Fig.~\ref{fig:impact}).

%% ─── 5. Discussion ───────────────────────────────────────────────────────────
\section{Discussion}

\subsection{When Does +Repo Help Most?}
Repository context provides the largest gains on bugs rooted in helper functions
imported by the crashed module (e.g., \texttt{swapped}, \texttt{none\_attr}).
For self-contained bugs the improvement is marginal. Practical heuristic:
enable \texttt{--repo} when the crash occurs in code that imports
project-defined modules.

\subsection{When Does +Git Help Most?}
Git context is most valuable for regressions introduced by a recent commit.
\texttt{off\_by\_one} and \texttt{recursion} showed the largest +Git gains,
consistent with bugs visible in \texttt{git diff HEAD\textasciitilde1}. For
bugs that predate many commits, git context adds noise rather than signal.

\subsection{Limitations}
\textbf{Keyword matching} is a proxy for semantic correctness; future work
should adopt LLM-as-judge evaluation.
\textbf{Fix correctness} checks only the original input vector.
\textbf{Source availability}: post-mortem analysis requires source files
at the paths recorded in the traceback.
\textbf{Single model}: all experiments use GPT-4o; cross-model benchmarking
is left to future work.

%% ─── 6. Conclusion ───────────────────────────────────────────────────────────
\section{Conclusion}
We presented ChatDBG-MTP, extending ChatDBG with post-mortem crash-log
analysis, import-graph and git context, and an automated fix-and-test repair
loop. On 8 benchmarks, +Repo+Git raises keyword coverage from 57\,\% to 89\,\%
and fix correctness from 50\,\% to 88\,\%, at a 38\,\% cost increase. The
bundled evaluation harness enables systematic comparison of context strategies
and forms a foundation for future LLM-assisted program repair research.
